If something isn't working, check here first.

Troubleshooting reference
all common errors
SymptomCauseFix

Render shows "Cannot GET /"public/index.html not foundMake sure the file is at public/index.html — not in the root or a different folder name
Deploy log: "Missing SUPABASE_URL"Environment variables not setIn Render dashboard → your service → Environment tab → add the two Supabase variables, then click Manual Deploy
/api/claims returns empty arrayRLS policy blocking readsGo to Supabase → SQL Editor → re-run the CREATE POLICY lines from supabase_setup.sql
PDF downloads but shows "6 dummy claims" onlyAPI not connectedVisit your-url/api/claims — if it returns dummy data, the Supabase env vars are missing or wrong
Render service "spins down" after inactivityFree tier sleeps after 15 minNormal behaviour on the free plan. First visit after sleep takes ~30 seconds to wake up. Upgrade to paid to avoid this.
Charts still look light/washed outBrowser cacheHard refresh with Ctrl+Shift+R (or Cmd+Shift+R on Mac)
Export PDF button does nothingjsPDF CDN failed to loadCheck browser console for errors. Hard refresh the page and try again. The jsPDF library loads from Cloudflare CDN.

How to update the dashboard in the future:

Go to your GitHub repo → navigate to the file you want to change → click the pencil icon (Edit) → make changes → click Commit changes. Render detects the push and automatically re-deploys within 2 minutes.
